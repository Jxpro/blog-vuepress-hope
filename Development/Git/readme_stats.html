                                            # 在自己的Vercel服务器部署GitHub Stats

                                            # 一、Fork 项目仓库

                                            首先fork项目github-readme-statsopen in new window到自己的GitHub下，然后修改/vercel.json文件才能部署成功，不然会提示部署错误，需要付费升级Pro计划

                                            • 修改maxDuration为1~10，推荐修改为10
                                            • destination用于设置Vercel域名重定向的url
                                            {
  "functions": {
    "api/*.js": {
      "memory": 128,
      "maxDuration": 30
    }
  },
  "redirects": [
    {
      "source": "/",
      "destination": "https://github.com/anuraghazra/github-readme-stats"
    }
  ]
}

                                            # 二、注册Vercel账号

                                            进入Vercel官网open in new window，使用GitHub进行注册

                                            # 三、导入项目

                                            找到Import Git Repository准备导入仓库

                                            点击Add GitHub Org or Account，选择Install授权访问仓库

                                            点击Import导入项目

                                            # 四、部署项目

                                            在GitHub里生成open in new window的token令牌，生成令牌时Select scopes可全部留空

                                            回到Vercel部署项目，选择不创建团队，点击跳过

                                            项目名称可自定义，其他内容保持不变

                                            添加名为PAT_1的环境变量，Value为刚才生成的Token

                                            点击deploy，等待约一分钟即可完成

                                            # 五、调用API

                                            [![Jxpro's GitHub stats](https://{domain}/api?username=jxpro&show_icons=true&theme=blueberry&include_all_commits=true)](https://github.com/Jxpro/github-readme-stats)
                                            • 详细参数说明见 readme.mdopen in new window
